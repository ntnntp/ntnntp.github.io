As a person who interested in IT, when I was in Grade 4 (P.4), I have a chance to learn coding HTML website from scratch. No Adobe Dreamweaver, no Wordpress, nothing— I coded in an ordinary Notepad on a Windows Vista machine. Like any other first code for dummies, I have managed to show "Hello World" in an Internet Explorer.

I learned how website works, how to host them as a local website (aka. local hosting) via XAAMP, integrating Joomla CMS into the website, and finding your own domain. Anyway at that time, I didn't have my own domain as it would not make so much sense for a newb kid to own his domain, so the website only worked in our local network. The website, obviously, doesn't look that good but I still managed to learn something most people at my age won't bother to learn.

Roughly four or five years later on, Saint Gabriel's College finally taught me to code in HTML, and I still remember that experience vividly. The instructor was Mr. Atithai, and he taught us "code once, use forever." Basically, it was a reminder for us to work efficiently by creating code or design templates for further duplication. Still it was a pure HTML with no tools involved— classic html, head, body things. Then finally in Grade 11 (Matthayom 5), we were taught Adobe Dreamweaver. This makes building website much more easier as it features Live Preview and editing from design. Though, the website we built is just a bunch of HTML files in the local machine; and no XAAMP or local hosting this time, just a open a file and, there you go, your web page. Clickable, with href (link) functions, but inaccessible anywhere else even in a networked computer.
I must mention that I have learned about one of many website builders out, too. I used it once it was a Flash website builder (quite some time before HTML5 is a real thing.) However, I did not quite like it very much as, first and foremost, it puts a sticky Created by *** banner on every single page of your website to promote its service, which is understandable— but it just made the whole website unprofessional and looks cheap. Not only that, it hosts your website on yourusername.***site.com domain and puts a pesky text "Proudly created by ***" at the footer of your website too. All of these cannot be removed unless you paid for its premium plan, which I believe starts around $8.50 or THB 255 per month— a small cost for startups and enterprises but quite a dealbreaker for students whose income are only from their parents, and this plan comes with only a year of free domain. Don't get me wrong though, that service is really really great for beginners wanting to create websites for free with no technical knowledges and wants a one-stop solution; it's just not for me.
Lately, I decided it is finally a time to have my own portfolio website. At first I made up my mind and decided to go with the above service. However, I came across Github Pages. The service allow us to host our HTML pages at no cost, and no banners, no created by xxx text— and it also provides free (dot)github.io domains, which, can be changed to any domain we owned for FREE. So I created my first web page using Tumult Hype, a HTML5 content creation tool.
The page loads just fine, but the responsive design seems to be the problem as it glitches around in various breakpoints. The smaller the screen, the more contents need to be scaled, and that makes elements jump around, misplaced, too small or too large, or even overflowed from visible viewport. This makes the whole process of creating my website a lot slower as more time is required to fix the bugs and glitches for various devices. Still, I just live with it for quite some time, until I finally decided to move on to better API and tools as I can't resort building the whole site for a month just to have it glitched on user's computer or even in front of HR's computer at a company interested in hiring me. That would be disastrous as seen below...

According to that, I stumbled around in Github, and found that Github has the Github Students Developer Pack. Essentially, bunch of companies sponsored this program and gave away their subscriptions and application licenses for free while you're a student. This includes a well-known Canva, Github Pro subscription, name.com domain registration, Unity, Microsoft Azure, and Bootstrap Studio which I am currently using to create this website from scratch.
It was the right decision for me to switch, as Tumult Hype wasn't meant to be used to create the whole site in the first place. Bootstrap Studio has made my workflow a lot faster and more intuitive while retaining the ability to code in any HTML, CSS, or even Javascripts when necessary.
Did I mention about CSS? Yes! I learned them by myself too. It was a trial-and-error learning but turns out it wasn't that hard at all. Also, I finally bought my own domain for THB 320 per year— a huge save for me for a website having dot com domains, perfect hosting server, decent Cloudflare DNS with HTTPS certificate (well, in short, huge save for really good website platform), and it only requires some DIY in the beginning.
In the time of writing these, I'm still finding ways to improve my website to make it load even faster, letting my friends and colleagues click around in my website to find any bugs and suggests any improvements, and willing to integrate my knowledge of UX/UI design I'm learning from different online courses into my website.

All in all, these will not be possible if I do not believe in myself that I can. I will admit that it was not easy, it was even a rough and bumpy road for me,
but in the end, it finally pays off...
